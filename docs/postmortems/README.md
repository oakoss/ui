# Postmortems

Public, blameless writeups of incidents, breaking changes, and significant bugs.

## When to write a postmortem

- Unintended breaking change shipped to consumers
- Security vulnerability disclosed
- Production incident affecting the docs site, registry, or MCP server
- Bug that caused consumer apps to break

## Format

- Filename: `YYYY-MM-DD-short-kebab-title.md`
- Use the template at [`_template.md`](_template.md)
- Blameless: focus on systems and processes, not individuals
- Public: postmortems are a trust signal for enterprise consumers

## Reference

IBM Carbon publishes postmortems openly. We follow the same model.
