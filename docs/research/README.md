# Research Notes

Findings from investigations that inform decisions but are not decisions themselves. Examples: comparative analyses of primitive layers, ecosystem surveys, performance studies, prior-art reviews.

## When to add research notes

- After investigating an option that we may revisit
- After comparing multiple alternatives where the trade-offs matter
- After surveying the ecosystem for missing pieces

## Format

- Filename: `short-kebab-title.md` (e.g. `primitive-layer-evaluation.md`)
- Include date, scope, and sources at the top
- Cite primary sources (URLs fetched, repo paths, npm registry data)

## Why this exists

Future contributors should be able to see "we looked at X and chose Y because Z" without re-litigating. Research notes prevent repeat investigations.
